# TaskyHub Infrastructure - Complete Setup Guide

This repository contains fully parametrized infrastructure code for deploying TaskyHub SaaS application for multiple customers.

## 🧪 Local Testing

For development and testing, see [LOCAL_RUN.md](docs/LOCAL_RUN.md) for instructions on running TaskyHub locally using Docker Compose.

## 📋 Quick Start

```bash
# 1. Copy and configure credentials
cp config/config.env.example .env
# Edit .env with your AWS and domain details

# 2. Make script executable
chmod +x scripts/spinup.sh

# 3. Run single deployment command
./scripts/spinup.sh
```

**PowerShell Alternative (Windows):**
```powershell
Copy-Item config/config.env.example .env
# Edit .env

.\scripts\spinup.ps1
```

## 🔧 Complete Setup Steps

### Step 1: Prerequisites

1. **AWS Account**
   - Create AWS account at https://aws.amazon.com
   - Create EC2 key pair and download .pem file
   - Get AWS Access Key ID and Secret Access Key from IAM

2. **Install Tools**
   - Terraform: https://developer.hashicorp.com/terraform/downloads
   - Ansible: https://docs.ansible.com/ansible/latest/installation_guide/
   - AWS CLI: https://aws.amazon.com/cli/

3. **Namecheap Domain**
   - Purchase domain from Namecheap
   - Create subdomain (e.g., tasky.amroth.life)

### Step 2: Configure Credentials

```bash
# Copy the example configuration
cp config.env.example .env

# Edit .env with your values
nano .env  # or use your preferred editor
```

**PowerShell Alternative (Windows):**
```powershell
Copy-Item config.env.example config.env
```

**config.env Template:**
```env
AWS_ACCESS_KEY_ID=your_access_key
AWS_SECRET_ACCESS_KEY=your_secret_key
AWS_REGION=us-east-1
KEY_PAIR_NAME=your-ec2-key-pair
CUSTOMER_NAME=tasky
UI_DOMAIN=taskyhub.xyz
UI_PORT=8080
API_DOMAIN=taskyhub.xyz
API_PORT=4000
SSH_KEY_PATH=/home/username/.ssh/your-key.pem
N8N_PASSWORD=YourN8nPassword123!
ADMIN_PASSWORD=YourAdminPassword123!
USER_PASSWORD=YourUserPassword123!
```

### Step 3: Setup AWS CLI

```bash
aws configure
# Enter your credentials when prompted
```

### Step 4: Deploy Infrastructure

**Single Command Deploy (Linux/Mac):**
```bash
chmod +x spinup.sh
./spinup.sh
```

**PowerShell Deploy (Windows):**
```powershell
.\spinup.ps1
```

**What This Does:**
- ✅ Initializes Terraform
- ✅ Provisions AWS EC2 instance with security groups
- ✅ Retrieves instance public IP
- ✅ Updates Ansible inventory automatically
- ✅ Runs Ansible playbook to configure server
- ✅ Installs Docker, nginx, certbot
- ✅ Deploys n8n container with your configuration
- ✅ Configures SSL certificate

### Step 5: Update Namecheap DNS

After deployment completes with instance IP, update DNS:

1. Go to Namecheap Dashboard
2. Find your domain (amroth.life)
3. Advanced DNS → Add A Record:
   - **Host:** @ for root domain, or use only the subdomain label for a subdomain.
     - Example: `customer1` to use `customer1.tasky.amroth.life`
     - Do not enter the full domain name in the Host field.
   - **Type:** A
   - **Value:** [Instance IP from deployment output]
4. Wait 5-10 minutes for DNS propagation

> Note: `terraform/user_data.sh` bootstraps the server environment. `ansible/playbook.yml` is the deployment workflow that installs UI and starts n8n.

### Step 6: Access Application

Once DNS propagates:
- **Dashboard:** `http://mummy.taskyhub.xyz`
- **Analytics (TaskyHub):** `http://mummy.taskyhub.xyz/analytics`
- **AE (n8n) Workflows:** `http://ae.mummy.taskyhub.xyz`

## 🚀 Adding New Customers

# To add a new customer (e.g., acme.taskyhub.xyz):

```bash
# Copy and rename config
cp .env config-acme.env

# Edit with new customer details
# CUSTOMER_NAME=acme
# UI_DOMAIN=taskyhub.xyz
# [update other values]

# Deploy
./spinup.sh config-acme.env
```

**PowerShell Alternative:**
```powershell
Copy-Item .env config-acme.env
# Edit config-acme.env

.\spinup.ps1 -ConfigFile config-acme.env
```

Each customer gets their own:
- EC2 instance
- Docker containers (`/opt/acme/n8n`)
- UI dashboard
- SSL certificate
- n8n workflows and data

## ⚙️ Customization

### Update Variables

Edit `terraform/variables.tf` and `ansible/vars.yml` to add more customizable options.

### Modify n8n Configuration

Edit `ansible/templates/n8n/docker-compose.yml.j2` to change n8n settings:
```yaml
environment:
  - N8N_HOST={{ domain_name }}
  - N8N_BASIC_AUTH_PASSWORD={{ n8n_admin_password }}
  # Add more environment variables here
```

### Modify Dashboard

Edit `ansible/templates/ui/dashboard.html` to customize the dashboard UI.

### Update Nginx Configuration

Edit `ansible/templates/tasky.conf.j2` for nginx settings.

## 🔐 Security

- Change all default passwords in config.env
- Restrict SSH access in AWS Security Group (add your IP only)
- Use strong passwords (minimum 12 characters)
- Enable 2FA on AWS and Namecheap
- Regularly backup n8n data: `/opt/[customer_name]/data/n8n`
- Update n8n and Docker regularly

## 🐛 Troubleshooting

### SSL Certificate Issues
```bash
# SSH into instance
ssh -i your-key.pem ubuntu@instance-ip

# Check certificate status
sudo certbot certificates

# Renew manually
sudo certbot renew --force-renewal
```

### n8n Not Starting
```bash
# Check container status
docker ps -a

# View logs
docker logs [customer_name]_n8n

# Restart container
cd /opt/[customer_name]/n8n
docker compose restart
```

### Ansible Connection Issues
```powershell
# Verify SSH key permissions
icacls $env:SSH_KEY_PATH /inheritance:r /grant:r "%USERNAME%:F"

# Test SSH connection
ssh -i $env:SSH_KEY_PATH ubuntu@instance-ip
```

## 📝 Notes

- Each customer deployment creates separate infrastructure
- Instance IP must be configured in Namecheap DNS
- First deployment may take 5-10 minutes
- SSL certificates are auto-renewed
- n8n data persists in `/opt/[customer_name]/data/n8n`
- All credentials must be changed before production
