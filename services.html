Platform

Core capabilities for the TaskyHub SaaS product.

TaskyHub is built to manage workflow automation across customer workspaces without leaking secrets, mixing provider credentials, or losing tenant context.

Workspace-scoped integrations

Store client IDs, API keys, webhooks, tokens, and provider settings per workspace.

  • Google OAuth and API-key based providers
  • Active/inactive provider controls
  • Connection testing before activation

Secrets and credential profiles

Support encrypted secret storage and reusable credential references for builders and operators.

  • Masked secret hints in admin views
  • Workspace-level credential ownership
  • Profile sync for supported providers

AI workflow generation

Generate workflow definitions and use model providers inside automation steps.

  • Prompt-driven workflow creation
  • Provider-backed AI nodes
  • Saved data and endpoints inside flows

Runs and analytics

Track execution health, recent runs, usage, and AI consumption from the same app.

  • Per-workspace usage views
  • AI usage reporting
  • Run drill-down and status history

Saved data and endpoints

Persist reusable links, text, and endpoint definitions that builders can reference in Canvas.

  • Plain text or URL storage
  • Endpoint usage insight
  • Reusable workspace-level references

Role-aware operations

Expose the right controls to platform admins, customer admins, workflow developers, and support users.

  • Workspace-level permissions
  • Restricted admin-only settings
  • Tenant-safe operational controls
Delivery model

One product, many customer workspaces.

The platform is being built for multi-tenant SaaS delivery. That means integration settings, saved data, provider state, and access control must stay workspace-aware while shared infrastructure such as canonical OAuth callbacks remains stable.
